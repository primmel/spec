[[file-types]]
== File types

=== General

MMEL uses several file types, each serving a distinct purpose in the modelling
workflow.

=== MMEL model files (`.mmel`)

The primary model definition files. They use a custom DSL syntax documented in
<<syntax-reference>>.

A `.mmel` file contains a single model definition consisting of a header
(root, version, metadata) followed by declarations (roles, events, processes,
classes, registries, provisions, measurements, references, links, and
subprocess diagrams).

=== Mapping files (`.map`)

JSON files that define mappings between processes in different models. Used for
standards alignment and implementation traceability.

A mapping file maps *source model processes* to *target model processes*,
each with a description and justification.

[source,json]
----
{
  "@context": "https://bsi-ribose-smart.org",
  "@type": "MMEL_MAP",
  "id": "ISO14971",
  "mapSet": {
    "defaultns": { "id": "defaultns", "mappings": {} },
    "BS13485": {
      "id": "BS13485",
      "mappings": {
        "<source-process-id>": {
          "<target-process-id>": {
            "description": "",
            "justification": ""
          }
        }
      }
    }
  },
  "docs": {},
  "version": "v1.0.0-dev1"
}
----

The `id` field identifies the source model by its namespace. The `mapSet`
contains one entry per target model namespace, each holding a nested mapping
from source process IDs to target process IDs.

The `docs` field may contain template entries used by tooling. When present,
it follows the same structure as workspace state files, organizing document
data by registry.

[source,json]
----
{
  "@context": "https://bsi-ribose-smart.org",
  "@type": "MMEL_MAP",
  "id": "ISO14971",
  "mapSet": {
    "defaultns": { "id": "defaultns", "mappings": {}, "docs": {} },
    "BS13485": {
      "id": "BS13485",
      "mappings": {},
      "docs": {
        "RegistryID": {
          "id": "RegistryID",
          "docs": {}
        }
      }
    }
  },
  "docs": {},
  "version": "v1.0.0-dev1"
}
----

===== `.json` files (MMEL_MAP variant)

Files with the `.json` extension may also contain MMEL_MAP format data,
identical in structure to `.map` files. The `.json` extension is an acceptable
alternative to `.map` for mapping files. The `version` field is required in
MMEL_MAP files.

=== SDC files (`.sdc`)

Supplementary Document Content files. Contain clause-level content extracted
from source normative documents.

[source]
----
namespace#DPTM-doc
title#DPTM certification requirements
version#v1.0.0-dev1
###
1.1.1#Organisation shall have data protection policies and practices
1.1.1#approved by management,
5.1.3#NOTE This is a note.
5.1.3#EXAMPLE This is an example.
----

The file format consists of a header section (namespace, title, version)
followed by a `###` separator and a body section. Each line in the body uses
the format `clause-number#content-text`.

Lines prefixed with `NOTE`, `EXAMPLE`, or `DEFINITION` are classified
annotations that correspond to note types used in MMEL models.

=== Workspace state files (`.sws`)

JSON files containing runtime data state -- stored documents within data
registers. These represent the actual data records that have been populated
through process execution.

[source,json]
----
{
  "docs": {
    "RiboseCrimson": {
      "id": "RiboseCrimson",
      "store": {
        "TestingRegistry": {
          "id": "TestingRegistry",
          "docs": {
            "6400790380169285": {
              "id": 6400790380169285,
              "name": "Happy first document",
              "attributes": {
                "string": "yes",
                "boolean": "False"
              }
            }
          }
        }
      }
    }
  },
  "version": "v1.0.0-dev1"
}
----
