[[state-machine-syntax]]
== State machine syntax

=== General

A state machine declaration uses the `state_machine` keyword followed
by an entity name (matching a `class#data`) and a brace-delimited body.

=== Syntax

[source,mmel]
----
state_machine <EntityName> {
  initial <StateName>                               <1>

  states {                                          <2>
    <StateName>
    <StateName>
    // ...
  }

  transition <From> -> <To>                         <3>
    [action <ActionName>] {
    guard "<OCL predicate>"                         <4>
    cascade <EntityName> {                          <5>
      where "<OCL predicate>"
      set {
        <field>: <value>
      }
    }
    reference { <ReferenceID> }                     <6>
  }
}
----

<1> Initial state. Required.
<2> State list. Optional (inferred from transitions if omitted).
<3> Transition declaration. Zero or more.
<4> Optional guard predicate.
<5> Optional cascade block. Zero or more per transition.
<6> Optional normative references.

=== Properties

[cols="1,1,3"]
|===
|Property |Required |Description

|`initial`
|yes
|The starting state for new entities.

|`states`
|no
|Explicit state list. Inferred from transitions if omitted. Recommended
for documentation.

|`transition`
|no
|Transition declaration. Zero or more.

|===

=== Binding to an entity

The `<EntityName>` after `state_machine` must match a `class#data`
declared elsewhere in the model. The class is expected to have a
`status` field that holds the current state.

The parser:

1. Collects all `class#data` declarations
2. For each `state_machine <EntityName>`, validates that a matching
   class exists
3. Emits a warning (not error) if the class lacks a `status` field

=== Example: TestReport lifecycle

[source,mmel]
----
state_machine TestReport {
  initial DRAFT

  states {
    DRAFT
    SUBMITTED
    UNDER_REVIEW
    ACCEPTED
    REJECTED
  }

  transition DRAFT -> SUBMITTED action lab_submits {
    cascade FormInstance {
      where "testReportId = ${this.id}"
      set { status: "LOCKED" }
    }
  }

  transition SUBMITTED -> UNDER_REVIEW action ia_starts_review {
  }

  transition UNDER_REVIEW -> ACCEPTED action ia_accepts {
    guard "not some(FormInstance where testReportId = ${this.id} and result = 'FAIL')"
  }

  transition UNDER_REVIEW -> REJECTED action ia_rejects {
    guard "some(FormInstance where testReportId = ${this.id} and result = 'FAIL')"
  }
}
----
